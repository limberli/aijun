=========
Changelog
=========

Все значимые изменения проекта QA Junior AI Tool.
Формат основан на `Keep a Changelog <https://keepachangelog.com/ru/1.0.0/>`_,
проект придерживается семантического версионирования.


v0.2-beta — 2026-06-11
======================

Релиз, нацеленный на приближение качества генерации к GPT-4 и на удобство
локального/облачного запуска и анализа.

Added (добавлено)
-----------------
- **Двухпроходная генерация (Planner).** Сначала извлекается чек-лист функций
  требования, затем тест-кейсы генерируются по ним батчами; таблицы склеиваются
  со сквозной нумерацией ``TC-NNN`` и дедупликацией. Пробивает потолок одного
  прохода: и покрытие, и детализация держатся вместе (16 кейсов × 10 шагов против
  ~5 × 10 ранее). Компоненты: ``PlannerService``, ``TableMerger``,
  ``ResilientChatClient``. Настройки — ``qa.planner.*``.
- **Загрузчик/парсер .docx (Loader/Parser).** Детерминированное извлечение текста
  из документов требований через Apache POI (абзацы + таблицы). Эндпоинт
  ``POST /api/extract-text`` и кнопка «Загрузить .docx» в UI.
- **Переключатель анализа рисков.** Risk Analysis теперь опционален и по умолчанию
  выключен (флаг ``metadata.riskAnalysis`` / ``AnalysisRequest.includeRiskAnalysis``);
  в UI добавлен тумблер.
- **Локальный GPU-стек.** ``docker-compose-qwen.yml`` — Ollama + Qwen2.5 с
  ускорением на NVIDIA (по умолчанию ``qwen2.5:14b``, переопределяется
  ``OLLAMA_MODEL``).
- **Персистентное логирование запусков.** Файловые JSON-логи всех сервисов;
  ``./logs/cloud`` для облачного режима и ``./logs/local`` для локального.
- **Документация.** ``docs/LOCAL_GPU.md`` (запуск на RTX 4090 +
  топ проблем и решений) и ``logs/README.md``.

Changed (изменено)
------------------
- **Модель по умолчанию (Groq).** ``llama-3.1-8b-instant`` →
  ``llama-3.3-70b-versatile`` — соблюдение формата и сложной инструкции вышло на
  уровень GPT-4 для задачи генерации тест-кейсов.
- **Промпт генерации (``qa-prompts.yml``).** Жёсткий 5-колоночный формат с
  обязательной непустой колонкой «Техника-тест дизайна»; few-shot-пример; правило
  покрытия (каждая функция — отдельный кейс, ориентир 8–12); детализация шагов как
  независимое от количества требование; парные негативные кейсы; требование реально
  применять технику в шагах, а не только в заголовке.

Fixed / Infrastructure
----------------------
- **Back-off по rate-limit (429).** ``ResilientChatClient`` соблюдает подсказку
  провайдера «try again in Xs», поэтому серия вызовов двухпроходной генерации
  переживает лимит токенов-в-минуту бесплатного тарифа Groq.
- **Тесты.** Добавлены юнит-тесты ``TableMerger`` и парсинга планнера, тесты
  парсера .docx, интеграционные тесты вкл/выкл анализа рисков; ``QaPromptBuilderTest``
  обновлён под 5-колоночный заголовок.


v0.1-beta — 2026-06-10
======================

Первый бета-релиз: мультиагентная система анализа требований на основе протокола
A2A (Agent-to-Agent).

Added (добавлено)
-----------------
- **Мультимодульное приложение** Spring Boot 3 + Spring AI: ``common``,
  ``host-orchestrator`` (:8080), ``tester-agent`` (:8081), ``analyst-agent`` (:8082).
- **Host Orchestrator.** ``POST /api/analyze`` — параллельный fan-out документа на
  двух агентов через ``CompletableFuture``, агрегация и сохранение диалога
  (H2/PostgreSQL). Эндпоинты ``GET /api/history``, ``GET /api/agents``,
  A2A-discovery ``GET /.well-known/agent-card.json``, Swagger UI.
- **Tester Agent.** Генерация структурированных тест-кейсов через Ollama/Spring AI;
  параметры генерации (тип кейса, техника, детализация) задаются каталогом
  ``qa-prompts.yml`` и приходят из UI.
- **Analyst Agent.** Анализ рисков и неоднозначностей требований.
- **Протокол A2A** поверх JSON-RPC 2.0 (``tasks/send``), обнаружение агентов по
  agent card при старте.
- **Отказоустойчивость.** Цепочка Resilience4j: Circuit Breaker → Time Limiter →
  Retry с fallback в ``AgentUnavailableException``.
- **Веб-интерфейс** ``a2a-ui`` на Next.js 15 + MUI 7.
- **Развёртывание.** ``docker-compose.yml`` (Ollama ``qwen2.5:7b`` с healthcheck),
  ``docker-compose-simple.yml`` (``phi3:mini``), профиль Groq для облачного инференса.
- **Тесты.** Интеграционные тесты пайплайна на WireMock (без реального Ollama).
